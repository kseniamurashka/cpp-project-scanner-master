# C++ Project Scanner - Лабораторная работа

## Постановка задачи
Реализовать консольную утилиту `cpp-project-scanner`, которая анализирует указанную директорию (по умолчанию — текущую) и генерирует отчет о C++ проекте.

**Утилита должна:**
- Рекурсивно обходить все файлы и поддиректории в заданной директории
- Идентифицировать C++ исходные файлы (по расширениям `.cpp`, `.cc`, `.cxx`, `.h`, `.hh`, `.hpp`, `.hxx`)
- Собирать статистику:
  - Общее количество C++ файлов
  - Общее количество строк кода и размер в байтах
  - Самый "тяжелый" C++ файл (наибольший размер)
  - Самый "длинный" C++ файл (наибольшее количество строк)

Формат вывода:
```txt
Total C++ files: 3
Total lines of code: 0
Total size: 168 bytes

Largest file: programm.cpp (132 bytes)
Longest file: data.h (0 lines)
```

## Требования к реализации
- Использовать фичи C++17/20:
  - `std::filesystem`
  - `std::optional` 
  - Ranges (C++20)
- Рекомендуется использовать `std::format` (C++20)
- Обрабатывать ошибки файловой системы
- Создавать читаемый вывод статистики

## Способы сборки

### Рекомендуемый способ: Docker

```bash
# Запускаются тесты и вывод на тестовых проектах (build.sh):
docker build -t cpp-scanner .
docker run -it -v ./src:/workspace/src cpp-scanner
```

### Иначе - обычный cmake
```bash
cmake -B build -S .
cmake --build build
cd build
./tests/run_tests  # запуск тестов
./cpp-project-scanner .  # запуск программы
```

## Инструкция по сдаче работы

1. Сделайте **форк** данного репозитория (если делаете его приватным, добавьте @avolkov00 как участника)
2. Выполните разработку в **вашем форке**
3. Реализуйте программу согласно требованиям в директории `src/`
4. Убедитесь, что все тесты проходят: `./build.sh`
5. Сдайте **ссылку на ваш форк** в СДО (дедлайн - 22 декабря)

**Важно:** 
- Не изменяйте файлы в директориях `tests/`, `cmake/`, а также `Dockerfile`, `CMakeLists.txt`, `build.sh`. Иначе не сработают автотесты у преподавателся и ему придется смотреть Вашу работу отдельно ☹
- Все ваши исходные файлы должны находиться в директории `src/`
- Вы можете создавать любую структуру файлов внутри `src/`

## Проверка на плагиат

Работы проверяются на оригинальность. За плагиат (копирование кода у других студентов) лабораторная не засчитывается.

## Теоретическая справка (кратко)

- **`std::filesystem` (C++17):** операции с файловой системой
- **`std::optional` (C++17):** опциональное значение
- **Ranges (C++20):** выразительная работа с диапазонами
- **`std::format` (C++20):** современное форматирование строк


## Получение помощи
Если возникли проблемы с запуском:

2. Убедитесь, что все файлы в правильных директориях
3. Запустите `./build.sh` для диагностики
4. Если проблемы остаются, обратитесь к преподавателю
